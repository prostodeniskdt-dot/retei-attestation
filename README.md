# Рётэй — Веб‑приложение для аттестации

Одностраничное приложение (SPA) для проведения аттестации сотрудников по меню «Рётэй». Работает прямо из браузера, сервер не требуется.

## Что умеет
- **Одна общая аттестация** по всем вопросам.
- **Три правдоподобных неверных варианта** у каждого вопроса, порядок вариантов случайный.
- **Кнопка «Начать»** запускает таймер на **30:00**. Состояние таймера и теста сохраняется в `localStorage`, так что обновление страницы **не** «сбрасывает» аттестацию.
- **Кнопка «Закончить»** завершает аттестацию и показывает подробный **отчёт** (верные/неверные ответы и процент успеха). Если прошло больше 30 минут — в отчёте явно указано, что аттестация **провалена** по времени.
- **Поделиться отчётом**: кнопка использует Web Share API, а также доступна печать в **PDF** (Сохранить как PDF). Используются системные шрифты с поддержкой кириллицы.
- **Небольшой личный кабинет**: экран входа с именем сотрудника и должностью; эти данные попадают в отчёт.
- **Чёрно‑жёлтый стиль** (можно настроить в `styles.css`).

Источник вопросов — прикреплённый файл пользователя «Аттестация по меню Рётэй - Вопросы для аттестации.pdf». fileciteturn0file0

## Быстрый старт (локально)
1. Откройте папку `app` и запустите файл `index.html` двойным кликом — браузер откроет приложение.
   - Для стабильной работы с `localStorage` лучше открыть через небольшой локальный сервер, например:
     - Python: `python3 -m http.server 8000` → затем перейти на http://localhost:8000/app/
2. Введите имя и должность → «Войти».
3. Нажмите «Начать» — запустится таймер и появится первый вопрос.
4. После прохождения нажмите «Закончить», затем «Скачать PDF» (печать) или «Поделиться».

## Деплой на GitHub Pages (рекомендуется)
1. Создайте новый репозиторий на GitHub (например, `retei-attestation`).
2. Содержимое из этой папки **целиком** положите в корень репозитория.
3. Включите GitHub Pages:
   - Settings → Pages → Source: **Deploy from a branch**.
   - Branch: **main** / **master**, Folder: **/** (root), Save.
4. Через ~30–60 секунд приложение будет доступно по адресу GitHub Pages.

## Подробная инструкция по созданию и установке репозитория

### Вариант A — через браузер
1. На GitHub нажмите **New** → введите имя репозитория, например `retei-attestation` → **Create repository**.
2. Нажмите **Add file** → **Upload files** → перетащите содержимое этой папки (включая `app/`) → **Commit**.
3. Откройте Settings → Pages → настройте GitHub Pages (см. выше).

### Вариант B — через Git
```bash
# Замените YOUR-USERNAME на ваш логин GitHub
git init retei-attestation
cd retei-attestation

# Скопируйте все файлы из архива в текущую папку (чтобы были README.md и папка app/)
git add .
git commit -m "Initial commit: Рётэй аттестация"

git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/retei-attestation.git
git push -u origin main
```

Далее включите GitHub Pages (Settings → Pages).

## Про печать в PDF и кириллицу
Отчёт формируется как отдельный блок HTML с печатными стилями (`print.css`). При нажатии «Скачать PDF» вызывается **диалог печати браузера**, где можно выбрать «Сохранить как PDF». Браузеры корректно сохраняют кириллицу, т.к. используется `system-ui` шрифт с полным покрытием. При необходимости вы можете подключить собственный WOFF2‑шрифт с кириллицей в `styles.css` (смотрите комментарии в файле).

## Структура
```
app/
  index.html      # Приложение
  styles.css      # Тема: чёрно‑жёлтая
  print.css       # Печатные стили для PDF отчёта
  app.js          # Логика: вход, таймер, тест, отчёт, share/print
  data/questions.json  # Вопросы + варианты
  assets/logo.svg
```

## Настройка вопросов
Изменяйте/добавляйте вопросы в `app/data/questions.json`. Формат:
```json
{
  "id": "q001",
  "text": "Текст вопроса…",
  "answers": [
    { "text": "Правильный", "correct": true },
    { "text": "Неправильный 1", "correct": false },
    { "text": "Неправильный 2", "correct": false },
    { "text": "Неправильный 3", "correct": false }
  ]
}
```
Порядок вариантов рандомизируется при старте.

## Без «уловок» после старта
- При нажатии «Начать» сохраняется отметка времени `startedAt` в `localStorage`. Обновление страницы, повторный вход, закрытие/открытие вкладки **не** позволят сбросить таймер или стартовать заново.
- Завершить можно только кнопкой «Закончить». Если фактическое время превышает 30 минут — отчёт помечается как «Провалено по времени».
